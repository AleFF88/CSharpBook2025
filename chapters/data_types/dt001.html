Ссылочные типы данных (reference types) и значимые типы данных (value types). CLR поддерживает две разновидности типов: ссылочные и значимые. Большинство типов являются ссылочными, однако зачастую мы используем именно значимые типы. Переменные значимого типа хранятся в стеке или непосредственно внутри объектов, если являются полями. УТОЧНИТЬ Переменная такого типа хранит само значение, а не ссылку на него. Когда значение копируется, создаётся полная независимая копия. В переменных ссылочных типов хранятся ссылки на их данные (объекты). При создании такой переменной выделяется память в управляемой куче, а оператор C# возвращает адрес этой памяти. Важно понимать, что неосторожное обращение с переменными ссылочного типа может существенно снизить производительность программы и привести к ошибкам, которые не удаётся обнаружить на этапе компиляции. При копировании переменной копируется ссылка, а не сам объект — поэтому две переменные могут указывать на один и тот же объект. Дело в том, что две такие переменные могут указывать на один и тот же объект в памяти. В результате любые изменения, внесённые через одну переменную, автоматически отразятся и на другой, поскольку обе ссылаются на одну и ту же область памяти. Простое присваивание одной ссылочной переменной другой не создаёт копию объекта, а лишь дублирует ссылку на него. Кроме того, такие переменные размещаются в управляемой куче, что влечёт за собой дополнительную нагрузку на систему. После того как объекты становятся недоступными, они автоматически удаляются посредством сборщика мусора, что также требует ресурсов. Для ускорения обработки простых данных существует другой тип — значимые типы. Экземпляры таких типов обычно размещаются в стеке потока. Даже если они встроены в объект ссылочного типа, они всё равно хранятся в стеке. В этом случае переменная не содержит указателя на экземпляр, а хранит данные непосредственно. Кроме того, такие переменные не подвергаются сборке мусора, что снижает нагрузку на управляемую кучу и сокращает количество циклов уборки мусора во время работы приложения. Значимый тип не может выступать в роли базового для других типов — ни ссылочных, ни значимых. Нельзя указать такие типы, как Boolean, Integer и другие значимые типы, в качестве базовых при описании нового типа. Примитивные значимые типы: bool — логический тип (true или false) byte — 8-битное беззнаковое целое sbyte — 8-битное знаковое целое char — символ Unicode (16 бит) short — 16-битное знаковое целое ushort — 16-битное беззнаковое целое int — 32-битное знаковое целое uint — 32-битное беззнаковое целое long — 64-битное знаковое целое ulong — 64-битное беззнаковое целое float — 32-битное число с плавающей точкой double — 64-битное число с плавающей точкой Другие значимые типы: enum — перечисления (перечисляемые константы) struct — структуры — пользовательские типы, которые объединяют несколько значений УТОЧНИТЬ Основные ссылочные типы: object — базовый класс для всех типов в C#, все типы наследуются от него class — классы, которые описывают объекты с состоянием и поведением interface — интерфейсы, задающие контракты (наборы методов и свойств) без реализации delegate — делегаты, объекты, которые ссылаются на методы (аналог указателей на функции) array — массивы, коллекции элементов одного типа string — строки — специальные ссылочные типы для работы с текстом (неизменяемые) ДОДЕЛАТЬ Просмотри рихтер стр 152 там есть пояснения и рисунки на ней! дальше тоже интересное описание отдельное про упаковку и распаковку
