Приведение типов в C#

Неявное приведение типов (implicit)

Происходит автоматически компилятором, если преобразование безопасно:

int a = 10;
double b = a;               // Неявное приведение int → double
Console.WriteLine(b);       // 10.0

Здесь int → double — безопасно: вещественный тип способен представить целое число без потерь.

Явное приведение типов (explicit)

Нужно, если возможна потеря точности, переполнение или некорректные данные:

double x = 9.7;
int y = (int)x;         // Явное приведение double → int
Console.WriteLine(y);   // 9

Здесь дробная часть отбрасывается, поэтому требуется указание типа (int).

Оператор is

Проверяет совместимость объекта с данным типом и возвращает булевское значение:

int a = 100;
bool res1 = a is string;    // False
bool res2 = a is int;       // True

Для null результатом будет false.

Оператор as

Пробует преобразовать объект к заданному типу. При невозможности — возвращает null:

object obj = "Текст для печати.";
string str = obj as string;
if (str != null)
{
   Console.WriteLine(str.ToUpper());   // ТЕКСТ ДЛЯ ПЕЧАТИ.
}

Ключевые моменты:

Оператор Назначение Ошибка при несовместимости
is Проверка типа Нет
as Безопасное приведение ссылочных типов Нет (возвращает null)
(Type) Явное приведение Да, InvalidCastException
